College Student Management REST API
📌 Project Overview

This project implements a Student Management REST API that allows a college to manage student records digitally. The API supports basic CRUD operations such as creating, reading, updating, and deleting student data.

The API is built using the built-in Node.js HTTP module without using frameworks like Express.js. This helps demonstrate how REST APIs work at a lower level.

All responses are returned in JSON format, and the server handles validation, error responses, and proper HTTP status codes.

🎯 Project Objectives

Build a RESTful API using Node.js.

Handle HTTP methods: GET, POST, PUT, DELETE.

Validate student data before storing it.

Return structured JSON responses.

Implement proper error handling.

⚙️ Technologies Used

Node.js

Built-in HTTP module

Postman (for API testing)

JSON for data storage (in memory)

🚀 How to Run the Project
1️⃣ Install Node.js

Download and install Node.js if it is not installed.

Check installation:

node -v
2️⃣ Clone the Repository
git clone <your-repository-link>

Navigate to the project folder:

cd student-api
3️⃣ Run the Server
node server.js

Server will start on:

http://localhost:3000
📡 API Endpoints
1️⃣ Create Student

POST /students

Creates a new student record.

Example Request Body:

{
 "name": "Ash",
 "email": "ash@test.com",
 "course": "CS",
 "year": 2
}

Example Response:

{
 "success": true,
 "message": "Student added",
 "data": {
  "id": 1,
  "name": "Ash",
  "email": "ash@test.com",
  "course": "CS",
  "year": 2
 }
}
2️⃣ Get All Students

GET /students

Returns all student records.

Example Response:

{
 "success": true,
 "data": [
  {
   "id": 1,
   "name": "Ash",
   "email": "ash@test.com",
   "course": "CS",
   "year": 2
  }
 ]
}
3️⃣ Get Single Student

GET /students/:id

Example:

GET /students/1

Response:

{
 "success": true,
 "data": {
  "id": 1,
  "name": "Ash",
  "email": "ash@test.com",
  "course": "CS",
  "year": 2
 }
}
4️⃣ Update Student

PUT /students/:id

Example:

PUT /students/1

Request Body:

{
 "year": 3
}

Response:

{
 "success": true,
 "message": "Student updated",
 "data": {
  "id": 1,
  "name": "Ash",
  "email": "ash@test.com",
  "course": "CS",
  "year": 3
 }
}
5️⃣ Delete Student

DELETE /students/:id

Example:

DELETE /students/1

Response:

{
 "success": true,
 "message": "Student deleted"
}
✅ Validation Rules

The API ensures:

All fields are required

Email must follow a valid format

Year must be between 1 and 4

ID is generated by the server

If validation fails, the server returns:

{
 "success": false,
 "message": "Validation error"
}
⚠️ Error Handling

The API handles common errors such as:

Invalid Route
{
 "success": false,
 "message": "Route not found"
}
Student Not Found
{
 "success": false,
 "message": "Student not found"
}
